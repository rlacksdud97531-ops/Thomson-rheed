## 0️⃣ 사전 준비 (최초 1회)

- [ ] **GitHub 계정** → https://github.com/signup (무료)
- [ ] **Git 설치** → https://git-scm.com/download/win
- [ ] **Streamlit Cloud 계정** → https://share.streamlit.io (GitHub으로 로그인)

## 1️⃣ 로컬에서 먼저 테스트

배포 전에 로컬에서 잘 돌아가는지 확인:

```bash
cd "C:\Users\rlack\OneDrive\바탕 화면\rheed_public"

# 가상환경 만들기 (옵션이지만 추천)
python -m venv .venv
.venv\Scripts\activate

# 의존성 설치
pip install -r requirements.txt

# 실행
streamlit run app.py
```

브라우저에서 http://localhost:8501 열어서:
- ✅ 이미지 업로드 잘 됨?
- ✅ 예측 결과 나옴?
- ✅ 확률 막대 표시됨?

OK면 다음 단계로.

## 2️⃣ GitHub 저장소 만들기

### 방법 A — 웹에서 (쉬움)
1. https://github.com/new 이동
2. **Repository name**: `rheed-classifier` (원하는 이름)
3. **Public** 선택 ✅ (Streamlit Cloud 무료 플랜은 Public 필수)
4. **Create repository** 클릭 → 저장소 URL 복사

### 방법 B — 명령줄로
```bash
cd "C:\Users\rlack\OneDrive\바탕 화면\rheed_public"

git init
git add .
git commit -m "Initial commit: RHEED classifier public app"

# GitHub에서 생성한 저장소 URL로 교체
git remote add origin https://github.com/YOUR_USERNAME/rheed-classifier.git
git branch -M main
git push -u origin main
```

> ⚠️ **주의**: `Thomson_5.keras` (61MB)도 푸시돼야 함. GitHub 단일 파일 제한은 100MB니까 OK. 푸시가 느릴 수 있음 (~1-2분).

---

## 3️⃣ Streamlit Cloud에 배포

1. https://share.streamlit.io 접속 → **Sign in with GitHub**
2. 우측 상단 **"Create app"** or **"New app"** 클릭
3. 폼 입력:
   - **Repository**: `YOUR_USERNAME/rheed-classifier`
   - **Branch**: `main`
   - **Main file path**: `app.py`
   - **App URL** (옵션): 원하는 서브도메인 (예: `rheed-cnn`) → `https://rheed-cnn.streamlit.app`
4. **Deploy!** 클릭

### 🕐 처음 배포는 5~10분 걸림
- 의존성 설치 (TensorFlow가 무거움)
- 모델 로딩

Streamlit Cloud 로그 화면에서 진행 상황 실시간으로 볼 수 있음. 완료되면 URL 나옴.

---

## 4️⃣ 공유!

`https://YOUR-APP.streamlit.app` 이 링크를 연구실 / 친구 / 학회 발표 슬라이드에 써도 됨.

- ✅ 24/7 접속 가능
- ✅ 내 PC 꺼도 살아있음
- ✅ 모바일에서도 접속 가능 (반응형)
- ✅ 무료

### ⏰ 유휴 Sleep 주의
10분 이상 아무도 접속 안 하면 "sleep" 상태가 됨. 누가 다시 접속하면 **10~20초 뒤에 깨어남** (자동). 깨어있는 동안은 즉시 응답.

## 🔄 업데이트는 어떻게?

로컬에서 코드 수정 → `git push` 하면 **자동 재배포** (1~2분).

```bash
# 코드 수정한 뒤
git add .
git commit -m "Update: 설명 추가"
git push
```

끝. Streamlit Cloud가 자동으로 감지해서 재배포해줌.

## 🛟 문제 해결

### 배포 실패 — "No module named X"
→ `requirements.txt`에 해당 패키지 추가 후 git push

### 앱이 시작 안 됨 — "Out of memory"
→ Streamlit Cloud 무료 티어 RAM 1GB 제한. EfficientNetB2는 안전하게 돌아감. 그래도 문제면 `app.py`에서 `tf.keras.backend.clear_session()` 추가

### 모델 파일이 너무 큼 (100MB 초과)
→ Thomson_5는 61MB라 OK. 만약 더 큰 모델 쓰게 되면 **Git LFS** 또는 **Hugging Face Hub**에 모델 올리고 앱에서 다운로드

### "This app is over the free tier limit"
→ 한 계정당 앱 여러 개 가능. 리소스 제한 있으니 하나만 유지
